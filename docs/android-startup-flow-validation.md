# Android Startup Flow Validation

## Summary

Android startup was validated on a physical Samsung SM-S928B running Android 16. The installed debug application launches through the Android splash screen, creates and retains its process, loads bundled Capacitor web assets, renders the Hebrew login UI, handles the root back action normally, and relaunches successfully. No Android fatal exception or JavaScript runtime crash was found.

The startup flow is **NO-GO** because the configured backend does not allow the Capacitor WebView origin. The phone can reach the backend at `http://192.168.1.10:8000` and receives HTTP 200, but an OPTIONS preflight from `Origin: https://localhost` to `/auth/login` receives HTTP 400 without `Access-Control-Allow-Origin`. Core API requests from the WebView are therefore blocked by CORS.

No application or native configuration was changed during this validation.

## API Connection

**Result: FAIL**

The Android build reads:

```text
VITE_API_URL=http://192.168.1.10:8000
```

This is a valid LAN target for the physical phone. The phone and backend host were on the same `192.168.1.x` network.

Direct device connectivity passed:

```text
GET API root -> HTTP 200
```

The browser security path failed. An OPTIONS request equivalent to the WebView login preflight used:

```text
Origin: https://localhost
Access-Control-Request-Method: POST
Access-Control-Request-Headers: content-type
```

Observed response:

```text
HTTP/1.1 400 Bad Request
Access-Control-Allow-Origin: missing
Access-Control-Allow-Methods: DELETE, GET, HEAD, OPTIONS, PATCH, POST, PUT
Access-Control-Allow-Headers: content-type
```

The backend is reachable, but its CORS configuration does not authorize the Capacitor origin. Login and other cross-origin API calls from the WebView are expected to be rejected by the browser.

This is an API configuration blocker, not an Android networking or routing failure.

## Known Issues

### Blocker

1. **Capacitor origin missing from backend CORS allowlist**
   - Backend is reachable from the device.
   - Preflight from `https://localhost` fails with HTTP 400.
   - Core authenticated and data API requests cannot reliably operate from the Android WebView.
   - Required action: add the exact Capacitor origin through the approved backend CORS configuration and revalidate preflight plus an actual app API request.

### Non-blocking

1. **Fresh Gradle validation was blocked by the execution environment**
   - The approval/usage gate rejected the command before Gradle started.
   - The installed APK was the verified ISSUE-199 artifact.
   - This does not explain the runtime CORS failure, but a fresh `assembleDebug` should be rerun when the environment permits.

2. **Google login load error is visible**
   - The login screen rendered an error indicating Google login could not load.
   - Startup remains usable for username/password UI.
   - OAuth functionality must be validated in its dedicated issue/PR.

## Final Verdict

**NO-GO**

The native Android startup shell is healthy: the app launches, the splash completes, the WebView renders bundled assets, the initial login route is usable, back/relaunch behavior is acceptable, and no crash occurs.

The complete startup flow is not ready for native feature work that depends on backend data because the backend rejects CORS preflight from `https://localhost`. This must be corrected and validated with a real WebView API request before the startup flow can receive a GO verdict.
